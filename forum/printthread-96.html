RE: setDAdisplay - hashah - 09-02-2018

The general idea is this : you do not know what type of value the user wants to store in the array.

Now, suppose the user asks you to display the array, how will you display it if you don't even know what's being stored in the array? You can't simply do something like this : "%d , %c".....

And that's where setDAdisplay function comes into place. User only knows what they are trying to store and so they should provide us a way to show their value.

So, in short :
1. setDAdisplay : receives the function and sets it to the property of DA struct.
2. displayDA : uses that property which is a function pointer and displays the value.


Answer to your question : displayChar, displayString is what user makes it and passes it to us. It could be anything for example : displayMyObject , displayMyStruct....

Hope this helps!
