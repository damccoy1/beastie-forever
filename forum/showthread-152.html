When you put a single value into a search tree, it is both key and value. In more realistic situations, the key is one piece of the value, usually a unique identifier. In the case of a GST, a value is inserted with a frequency count. So the generic value is the key, but there is more data associated with the key (i.e. the frequency count).

The TNODE object knows nothing about how the key is packaged with the value or even if they are the same thing. The BST tree is passed a comparator function that knows how to compare values. It understands which part of the value is the key. For example, the GST will insert values into the underlying BST. A GST value will consist of the generic value passed to insertGST and a frequency count. I use a private class to implement a GST value. That private class has a void * pointer to the generic value and an int frequency. It also has pointers to various functions that will be needed at some point.
