From [/url][url=http://beastie.cs.ua.edu/DS-intro/index_7.html]the circular array entry on the Beastie "Encyclopedia of Data Structures," why must we, when correcting the index of a CDA, either increment or decrement the index when calling the correctIndex() function? 

Furthermore, why must we decrement (while calling the correctIndex() function) the start index when inserting a value at the front and increment the end index when inserting at the back?

I've attached a picture showing what I'm referencing from the Encyclopedia.

RE: Correcting Index in CDA - lusth - 09-12-2018

If v is the current start index, then u = correctIndex(v-1) would have u "pointing to" the first unused slot prior to the beginning element. Likewise, if y is the end index, then x = correctIndex(y+1) would have x "pointing to" the first unused slot after the last element.
