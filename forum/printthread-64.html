Building a heap actually has O(n) complexity (read here). So the answer should just be Θ(n + m) as you could build a heap from all the elements together.

However, this merge algorithm makes me wonder if it would be Θ(log(n)*log(k)): https://link.springer.com/article/10.1007%2FBF00264229. The question does not specify if we are using an array-based or pointer-based implementation.

RE: Merging two heaps of size n and m, m < n takes how much time? - lusth - 09-06-2018

The Springer reference separates data movement and comparisons. I assume that in the article, the sized-k heap is the larger. So that makes the overall amount of work theta(k).

Making an array out of two heaps of size n and k and then building the heap takes theta(n + k) or theta(k) if k is the larger size.
