  Helpful Hint
Posted by: jwarnett - 10-04-2018, 10:17 PM - Forum: Project 1 - Replies (1)

Just to help people out, when you are doing -s, if the file you are reading from and writing to are the same, you will need to make sure to close the read version before you open the write version.  I did it the other way initially and got a segfault when testing against one of the student provided test cases.  I don't know if Dr. Lusth will be testing for the ability to handle both files being the same file, but doing this expands your functionality.  Good luck on y'alls projects.

  problems with random and -std=C99
Posted by: IsabelHerrick - 10-02-2018, 10:19 PM - Forum: Project 1 - Replies (1)

I have
#include <stdio.h>
#include <stdlib.h>

in my code, but when I use the random() function I get 
warning: implicit declaration of function ‘random’ [-Wimplicit-function-declaration]

I googled it, and a stackoverflow question gave me this answer:
man srandom says that the function is not part of C99 but part of POSIX.

How do I reconcile that with "You must implement your modules in C99 and you must use the -std=C99 option to gcc in your makefile."? Should we use rand() instead of random()? As far as I can tell the code still works, but it always compiles with a warning.

Edit: got my question answered. You have to put

extern long int random(void);
at the top of the file
