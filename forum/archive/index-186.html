I originally had "only union-by-rank" for question 23 (Which of the following Heuristics make sense for a linked-list implementation of a disjoint set?), but ended up changing my answer to "neither".

My reasoning was as follows: the rank of a set increases only when two sets of equal rank are unioned. So it could be possible to have a set of rank 2 with 4 nodes, and a set of rank 1 with 2000 nodes (if you keep unioning it with sets of rank 0). As a result, for the linked list implementation, it doesn't seem to make sense to use union by rank, but rather union by size, since union by rank (for a linked list) doesn't guarantee the minimum amount of work, while union by size does.

I believe both "only the second" and "neither" should be correct.
