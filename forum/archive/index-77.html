I have some helper functions that are declared in cda.h and da.h and defined in cda.c and da.c.  They are used by stack.c and queue.c and work perfectly on my system, but when I send it into the test server, I get an implicit declaration error.  Does anyone have any idea why that would be occurring even though they are declared and defined?

magarwal

09-14-2018, 02:39 PM
You can't change the public interface. If these are helper functions, they need to be "private" so they would stay in your .c files. So move the method prototypes for these functions from the .h files to the top of your .c files.

lusth

09-14-2018, 06:24 PM
Also private functions should be tagged with the static keyword in both the prototypes and the function definitions. Example

static int correctIndex(CDA *items,int i);
....
static int correctIndex(...) { ... }
