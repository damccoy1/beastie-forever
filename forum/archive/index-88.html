I have
#include <stdio.h>
#include <stdlib.h>

in my code, but when I use the random() function I get 
warning: implicit declaration of function ‘random’ [-Wimplicit-function-declaration]

I googled it, and a stackoverflow question gave me this answer:
man srandom says that the function is not part of C99 but part of POSIX.

How do I reconcile that with "You must implement your modules in C99 and you must use the -std=C99 option to gcc in your makefile."? Should we use rand() instead of random()? As far as I can tell the code still works, but it always compiles with a warning.
Edit: got my question answered. You have to put
extern void srandom(unsigned int);
extern long int random(void);
at the top of the file

Correct. Put:

long int random(void);
void srandom(unsigned int seed);

at the top of any file that uses random/srandom.
