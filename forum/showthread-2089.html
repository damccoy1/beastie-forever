Consider these operations to a CDA:

insert A at index 0
insert B at index 1
insert C at index 0
insert D at index 0

At this point, the array is full and looks like DCAB from the user's point of view but is ABDC internally with the start index at 2. Now, let's insert E at index 4. Since the array is full, we need to grow it. Using realloc, we would get ABDC++++, where the plusses represent unused slots. Inserting E at 4 slots beyond the start index would yield ABDC++E+, which is wrong in all kinds of ways. Instead you should allocate a new array, say temp, and then transfer elements from the old array:

temp[1] = getCDA(items,i);

This gives us DCAB++++ with the insert of E yielding DCABE+++, which is correct. Of course, the start index has been reset to zero in this process.
