Handling bad data - IsabelHerrick - 09-08-2018

If a parameter is passed that violates the assertions of a method, should we simply use 

assertion(expression) 

to end the program without any frills, or should we wrap it in an 

if(!expression) 
     print "you did something bad" 
     end program

or, combine both:

if(!expression)
     print "you did something bad"
assert(expression)

RE: Handling bad data - daweil - 09-09-2018

Use the assert function from assert.h (Your first option). It should provide enough detail.

Example:
I have x = 5, and then assert(x == 7). Executing my program results in the following output:

ex: ex.c:10: main: Assertion `x == 7' failed.
Aborted (core dumped)

The assert call was in fact on line 10.
