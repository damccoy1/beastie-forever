debugDA - what is the previous debug value? - hajames - 08-29-2018

From the project description: 

debugDA - The debug method sets an internal flag in the object to the given value. If the flag is greater than zero, the display method adds a comma (if needed) and the number of empty slots (in brackets) immediately after the last element. If the above array had a capacity of 8, the display would be [5,6,2,9,1,[3]]. An empty array with capacity 1 displays as [[1]] The method returns the previous debug value. 

The function signature is: 

Code:
int debugDA(DA *items,int level);

Two questions: 
1) What is the "previous debug value" that should be returned?
2) What is the "level" variable that is passed in with the function?

I'm pretty sure these two things are related, but I'm not sure exactly how or why they are necessary.

Thanks!

The debug value determines whether you display the number of available slots in your array.
