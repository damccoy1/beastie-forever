Sorry, if this was addressed in a previous post, but I couldn't find anything directly addressing my issue.

I'm going through the code for testing cda.c line by line, and I'm having trouble understanding what to do when shifting the indexes. Currently, I've edited the code to stop just before inserting the value at 3+i. Here is what my array looks like at this point:

|3|2| |4|
    ^   ^
    b    f

at i=0, the index to insert 1 is at 3, where the front of my array is. This is confusing me, as this would constitute inserting to the front of the array, result in the array print out as (1,4,3,2) instead of the intended (4,3,2,1).

The spec says that the previous value at the insertion spot is to be moved to the next higher spot, but higher in what way? If higher means pushing the previous value up one, inserting 4 when the array was just |3|2| would give me:

|4|2| |3|     then  |4|2|3|1|

 ^ ^                    ^ ^ 
  f  b                     f  b

which definitely isn't right. If higher means pushing the previous value down one, then the result would be

|4|3|2| |
 ^    ^
  f    b

which sets up the next insert all well and good, but looking ahead at the next level of inserts, constantly shifting to the right defeats purpose of the CDA and results in the wrong display as well.

tldr; In what regard is the "higher" in which you are shifting the index?

When inserting at the front of a CDA, it would be better to insert at the corrected index - 1. In this case, 1 would go at index 2 of your array, and you would change front to 2. There is a special case when front is already at 0.

You should make room for the new element by increasing the size first. Then, if you are inserting towards the front, you should decrement the start index and then shift elements to the left. Otherwise, just shift elements to the right.
