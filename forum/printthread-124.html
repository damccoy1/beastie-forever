Should the CDA index assertions 
Quote:
  • insertCDA - The index shall be valid. 

  • removeCDA - The index shall be valid. The size shall be greater than zero. 

  • getCDA - The index shall be greater than or equal to zero and less than the size. 

  • setCDA - The index shall be greater than or equal to -1 and less than or equal to the size.

be asserted before or after index correction? 
Asked slightly differently: should the "index" we are asserting be the given index value passed as an argument to these functions?

RE: CDA index assertions - dmwever - 09-03-2018

Beforehand for sure, for simplicity's sake.
You can never insert after size, and you can never insert before 0, even though where size and index actually are shift as items are added and removed.
While your CDA will probably look like this, |a|b|c|d| | | | | with a being the front and d being the back,
It can eventually, if you add to the back and remove from the front over and over, (like you will with the queue class) look more like this:
|c|d| | | | |a|b| with a being the front and d being the back.
If it does end up looking like that, it will be super complicated to process whether or not the index is within the legal range of values.

RE: CDA index assertions - magarwal - 09-03-2018

I think it would be after. The user only has access to the public interface and doesn't know anything about "index correction" or any implementation details, so for them, the index always refers to the corrected index. Of course, you don't actually need to "perform" any correction beforehand - you can just assume the index is corrected for the sake of assertion. The whole point of using those assertions is to validate inputs and inform the user about bad input.

TL;DR: Always look at it from the user's perspective.

From the user's perspective, if the CDA is (1,2,(2)) then inserting at index 4 is not valid. Checking the index before correction catches this. However, the corrected index in this case is 0, which is a valid index. So check the index first, then do any correction after asserting it is valid.
